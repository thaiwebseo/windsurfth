# Windsurf 2.0: แนะนำ Agent Command Center และ Devin ใน Windsurf

![Adaptive update cover](/images/blogs/windsurf-2-0-hero.png)

Windsurf เปิดตัวครั้งแรกในฐานะ agentic IDE ตัวแรก — เครื่องมือเขียนโค้ดที่คุณและ AI agent สามารถทำงานร่วมกันเคียงข้างกันในฐานโค้ดเดียวกันได้

ตั้งแต่นั้นมา Agent ต่างๆ ก็ทรงพลังมากขึ้น วิศวกรที่เก่งที่สุดไม่ได้จับคู่ทำงานกับ Agent เพียงตัวเดียวอีกต่อไป พวกเขารัน Agent นับสิบตัวพร้อมกันแบบคู่ขนานทั้งบนระบบคลาวด์และในเครื่อง (Local environments) เพื่อจัดการโปรเจกต์เดียวกันในส่วนที่ต่างกัน ในขณะที่ IDE ถูกสร้างขึ้นมาสำหรับการลงมือเขียนโค้ด (Hands-on work) แต่การจัดการฝูง Agent (Fleet of agents) นั้นเป็นงานที่แตกต่างออกไปอย่างสิ้นเชิง

สิ่งนี้นำไปสู่ปัญหาใหม่: ยิ่งคุณจัดการ Agent มากเท่าไหร่ คุณก็จะยิ่งถึงขีดจำกัดของความสนใจและความจำ (Working memory) ของคุณเองเร็วขึ้นเท่านั้น

 ![Adaptive model picker](/images/blogs/command-center.jpg)

## ภารกิจควบคุม (Mission control) ด้วย Windsurf agent sessions

**Windsurf 2.0** ออกแบบมาเพื่อทลายขีดจำกัดนั้น เราได้สร้าง **Agent Command Center** และนำ **"Devin"** ซึ่งเป็น Agent อัตโนมัติบนระบบคลาวด์ของเราเข้ามาใน Windsurf โดยตรง เพื่อให้คุณสามารถทำงานในระดับทีมได้โดยไม่ต้องมีคนอื่นมาช่วย

### The Agent Command Center (ศูนย์บัญชาการ Agent)
Agent Command Center คือพื้นที่ใหม่ภายใน Windsurf ซึ่งจะแสดง Agent ทุกตัวที่คุณกำลังรันอยู่ ทั้งบน Local และ Cloud ในมุมมองแบบ Kanban (คัมบัง) มุมมองเดียว ซึ่งจัดเรียงตามสถานะการทำงาน

มุมมองแบบ Kanban เป็นความตั้งใจในการออกแบบ เมื่อ Agent มีความสามารถมากขึ้น งานของวิศวกรจะเปลี่ยนจากการเขียนโค้ดไปสู่การสั่งการ (Directing work) คุณจำเป็นต้องเห็นว่า Agent ตัวไหนกำลังทำงานอะไรอยู่ งานไหนที่ติดปัญหา (Blocked) และงานไหนที่พร้อมสำหรับรอตรวจ (Ready for review) อินเทอร์เฟซนี้จึงออกแบบมาเพื่อสะท้อนสิ่งนั้นให้เห็นชัดเจน

 ![Adaptive model picker](/images/blogs/agent-editor-open.mp4)

ศูนย์บัญชาการนี้ไม่ได้เข้ามาแทนที่ IDE แต่มีการผสานรวมอย่างล้ำลึกกับฟีเจอร์เดิมของ Windsurf Editor ทำให้คุณสามารถทำการปรับแก้โค้ดในขั้นตอนสุดท้าย (Last mile edits) ด้วยตัวเองได้ตลอดเวลา

### Windsurf Spaces
Spaces คือวิธีที่คุณใช้จัดการงานใน Agent Command Center

Space จะจัดกลุ่มทุกอย่างที่เกี่ยวข้องกับงานหรือโปรเจกต์เฉพาะเจาะจงให้อยู่ในมุมมองเดียว ทั้งเซสชันของ Agent, PR (Pull Requests), ไฟล์ และบริบทต่างๆ ตัวอย่างเช่น Space ที่ชื่อว่า "Onboarding Flow Redesign" อาจจะมีเซสชัน Local หนึ่งตัวทำหน้าที่สร้างแบบจำลอง UI (Prototyping) และมีเซสชัน Cloud อีกสองตัวทำหน้าที่จัดการการเปลี่ยนแปลง API และเขียนเทสต์

 ![Adaptive model picker](/images/blogs/add-session-to-space.mp4)

Spaces สามารถจดจำบริบทได้ (Carry context) เมื่อคุณสร้างเซสชันใหม่ใน Space เซสชันนั้นจะได้รับสืบต่อข้อมูลทุกอย่างที่ Space นั้นรู้เกี่ยวกับโปรเจกต์ไปใช้งาน และเมื่อคุณกลับมาที่ Space นี้ มุมมองทุกอย่างก็จะยังคงอยู่เหมือนตอนที่คุณทิ้งไว้

การสลับระหว่าง Space คือการสลับระหว่างงาน แต่สิ่งที่เหนือกว่าในตอนนี้คือแต่ละงานจะมี "ทีม Agent" คอยทำงานอยู่ภายในนั้น

 ![Adaptive model picker](/images/blogs/user-switching-between-spaces.mp4)

### Devin ใน Windsurf
ควบคู่ไปกับ Agent Command Center ใหม่นี้ เราได้นำ **Devin** เข้ามาใน Windsurf โดยตรง

Devin คือ Agent วิศวกรซอฟต์แวร์อัตโนมัติที่ทำงานบนระบบคลาวด์ สามารถจัดการงานที่ซับซ้อนตั้งแต่ต้นจนจบ (End-to-end) — ทั้งการดีบัก (Debugging), การนำระบบขึ้นใช้งาน (Deployment) และการทดสอบระบบ (Testing) — และยังคงทำงานต่อไปได้แม้ว่าคุณจะพับหน้าจอแล็ปท็อปแล้วก็ตาม เนื่องจาก Devin มี Virtual Machine (VM) เป็นของตัวเอง พร้อมกับเดสก์ท็อป เบราว์เซอร์ และเครื่องมือคอมพิวเตอร์ครบชุด

 ![Adaptive model picker](/images/blogs/devin-in-windsurf.mp4)

ด้วยการมี Devin ใน Windsurf คุณสามารถมอบหมายงานขึ้นคลาวด์ได้โดยไม่ต้องออกจาก Editor คุณสามารถวางแผนงานกับ Local agent แล้วใช้เพียงคลิกเดียวเพื่อส่งไปให้ Devin นำไปดำเนินการต่อ จากนั้น Devin จะเปิดการทำงานบนเครื่องของตัวเองและเริ่มทำงาน คุณสามารถเลือกเขียนโค้ดในเครื่องต่อไป หรือพับแล็ปท็อปแล้วไปดื่มกาแฟชิลๆ ก็ได้

 ![Adaptive model picker](/images/blogs/devin-cloud-session.mp4)

เมื่อ Devin เปิด PR คุณสามารถตรวจสอบ (Review) ได้โดยตรงใน Windsurf ทั้งเช็กความแตกต่างของโค้ด (Diff), รันเทสต์ หรือส่งต่อให้ Local agent ของคุณเพื่อนำไปปรับแต่งในขั้นสุดท้าย

เป้าหมายของเราคือนำวงจรการพัฒนาซอฟต์แวร์ทั้งหมด — การวางแผน การมอบหมายงาน และการตรวจสอบ — มารวมไว้ในที่เดียวแบบไร้รอยต่อ
