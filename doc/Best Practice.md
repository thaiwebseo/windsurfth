# Best Practice ต้องรู้! ตั้งค่า Windsurf ยังไงให้ AI เขียน Code ตรงใจ 100% 🔥

การใช้งาน AI IDE อย่าง Windsurf ให้เกิดประสิทธิภาพสูงสุดนั้น นอกเหนือจากการสั่งให้เขียนโค้ดแล้ว ยังมีเทคนิคและแนวทางปฏิบัติ (Best Practices) ที่สำคัญเพื่อให้ AI เข้าใจโปรเจกต์และทำงานได้ตรงตามมาตรฐานที่เราต้องการ ดังนี้ครับ

## 1. ปูพื้นฐานด้วย Context ที่ชัดเจนผ่านไฟล์ `.windsurfrules`
AI จะทำงานได้ดีและฉลาดขึ้นอยู่กับข้อมูลที่เรามอบให้ การสร้างไฟล์กฏเหล็กให้ AI จึงสำคัญที่สุด
* สร้างไฟล์ชื่อ `.windsurfrules` ไว้ที่โฟลเดอร์ Root ของโปรเจกต์
* ใช้ไฟล์นี้ระบุข้อกำหนดต่างๆ เช่น เทคโนโลยีที่ใช้ (เช่น Next.js, TypeScript Strict Mode, Tailwind CSS, shadcn/ui) หรือมาตรฐานการเขียนโค้ด (เช่น ห้ามใช้ Class Component แบบเก่า หรือบังคับเขียนคอมเมนต์ภาษาไทย)
* **ข้อดี:** AI จะอ่านไฟล์นี้ทุกครั้งก่อนเริ่มงาน ทำให้ได้โค้ดที่ตรงมาตรฐาน ลดการใช้ Syntax เก่าๆ และที่สำคัญคือ "ไม่ต้องคอยพิมพ์สั่งซ้ำๆ" ในทุกๆ Prompt

## 2. ระบุเป้าหมายให้แม่นยำด้วยฟีเจอร์ `@mention`
เวลาต้องการให้ AI แก้ไขโค้ดหรือสอบถามข้อมูล ไม่ควรปล่อยให้ AI เดาเอง
* ให้พิมพ์เครื่องหมาย `@` ตามด้วยชื่อไฟล์ โฟลเดอร์ หรือเอกสารที่เกี่ยวข้อง (Document) ทุกครั้งที่สั่งงาน
* **ข้อดี:** ช่วยลดโอกาสที่ AI จะสุ่มหาไฟล์ผิด (Hallucination) และทำให้การแก้ไขตรงจุดรวดเร็วขึ้น

## 3. สั่งงานทีละสเต็ป (Divide and Conquer)
ข้อควรระวังคือ "อย่าสั่งงานหลายอย่างพร้อมกันใน Prompt เดียว"
* ควรแบ่งการสั่งงานเป็นทีละฟีเจอร์ เช่น สั่งให้สร้างหน้า Login ให้เสร็จก่อน จากนั้นค่อยสั่งให้ต่อ API Authentication
* **ข้อดี:** การทำทีละขั้นตอนจะช่วยลดข้อผิดพลาดและอาการหลอนของ AI ได้อย่างมาก 

## 4. ตรวจสอบโค้ดทุกครั้งก่อนกดยอมรับ (Review Before Accept)
แม้ว่า AI ในปัจจุบันจะเก่งมาก แต่เราในฐานะนักพัฒนาก็ยังต้องเป็นผู้ตรวจสอบขั้นสุดท้าย
* ก่อนจะกด Accept เพื่อรับโค้ด ควรอ่านและดูรายละเอียดเสมอว่า AI เข้าไปแก้ไขอะไรบ้าง 
* โดยเฉพาะอย่างยิ่ง หากมีการลบไฟล์หรือแก้ไขไฟล์ Configuration สำคัญๆ ต้องดูให้ละเอียดเป็นพิเศษ

## 5. ใช้ประโยชน์จาก Terminal สั่งรันและแก้บั๊กอัตโนมัติ
Windsurf ไม่ได้มีดีแค่เขียนโค้ด แต่ยังสามารถเชื่อมต่อกับ Terminal ได้ด้วย
* สามารถสั่งให้ AI รันโปรเจกต์ เขียนเทสต์ และวนลูปแก้โค้ดจนกว่าจะผ่านได้
* หากรันแล้วเจอ Error ใน Terminal ไม่ต้องเสียเวลาก๊อปปี้โค้ดมาแปะถาม แค่พิมพ์สั่งว่า "ช่วยดู Error ใน Terminal ให้หน่อย" AI จะเข้าไปวิเคราะห์และแก้ไขให้ทันที 

## 6. เลือกใช้โหมดการทำงานให้ถูกต้อง (Code, Ask, Plan)
Windsurf มีโหมดให้เลือกใช้งาน 3 รูปแบบหลัก เพื่อให้เหมาะกับวัตถุประสงค์การทำงาน:
* **โหมด Code:** ใช้สำหรับสั่งให้เขียนและแก้ไขโค้ดโดยตรง
* **โหมด Ask:** ใช้สำหรับสอบถามข้อมูลหรือขอคำแนะนำ (เช่น ขอแนวทางอัปเกรดระบบขึ้น Production) โดย AI จะตอบคำถามแต่ไม่เข้าไปยุ่งกับโค้ดของเรา
* **โหมด Plan:** ใช้เมื่อต้องการเริ่มต้นฟีเจอร์ใหม่ AI จะช่วยวิเคราะห์และแตกงานเป็นขั้นตอน (Step-by-step) ให้เราพิจารณาก่อน หากเรากดยืนยันแผนงาน AI ถึงจะเปลี่ยนเป็นโหมด Code เพื่อเริ่มเขียนจริง
